EOS Instruction Manual Guide

For Trader Account

 

 

Step-1: Access EOS URL ( Non-Azure Login ) -http://eosuat.southeastasia.cloudapp.azure.com/#/user/login

The below page will be displayed while accessing the EOS URL:

 

Step-2: Enter Trader username and password to the EOS system.

 


Step-3: After successful login to EOS system, the main EOS page will open which will display the Transaction Overview for Dashboard feature.

 


Navigation and Familiarisation

Step-1: There will be a utility menu situated on the left of the screen. This menu displays the pages the user has accesss to.

 

Step-2: A user can view their user profile by clicking on the user profile icon on the top right of the screen.

 

Step-3: Clicking on the User Profile page brings the user to their user profile page.

 

Step-4: Clicking on the Change Password page brings the user to a page to alter their login credentials.

 

For Dashboard Features:

Step-1: Click on the Dashboard feature to view Transaction details.

Note:

- A User can filter data displayed on the dashboard by selecting options available in the dropdown fields in the search bar of the page, followed by clicking on the “Search” button.

- Click on “Reset” button to reset the filter selection.

 

 

Step-2: Every EOS ID is linked to a detailed transaction view that displays all available timestamp data for the specified EOS transaction.

 

 

 

Step-3: Clicking on the “return to previous page” will return you to the dashboard feature.

Step-4: User can view the number of transactions with their details while clicking on “No of Transaction” section on total, closed, open and cancelled and their percentage of status is also displayed in bar.

 

Step-5: Users can use the hide view button to hide sections of the dashboard that they do not wish to display.

 


For Transaction Features:

Step-1: Click on the Transaction feature to view Summary of all Transaction.

 

Step-2: Click to “Expand” icon to view all the filter options as shown in below snapshot.

 

Step-3: User can select the required options through dropdown box appearing. Upon clicking on the “search” button, the summary of all transaction will be displayed for selected option.

Step-4: Click on “Reset” button to revert the changes made by selecting the options from dropdown box.

Step-5: Upon clicking the “EOS ID”, the Transaction details linked to that EOS ID will be displayed as shown below. E.g., “EOS ID” – “E22”

 

 

Step-6: Clicking on the “return to previous page” will return you to the main window page of that feature.

Step-7: To Print the details of the Transaction, user need to select the “EOS ID”, then click on “Print” button. User will be able to print the details.

Note:

- The Print button invokes the web browser’s built in print capabilities.

- If the selected table appears to overflow out of the PDF document, the user has to select a bigger paper size before printing the PDF.

- This behavior will be consistent across all PDF print functions of the EOS system.

 

 

Step-8: To download the details of the Transaction in csv, user need to select the “EOS ID”, then click on “CSV” button. User will be able to download the transaction details in csv format in their desktop.

 

Step-9: User can also sort in ascending, descending or unsort the sorted data for the fields that select for this activity. There are sorting button available with the columns name.

 

 


For Threshold Features:

Step-1: Click on Threshold Feature, user will see two dropdown option for “Summary” and “Triggered Alerts”. Click on “Summary” and below “Threshold Limit List” will appear.

 

Step-2: Click to “Expand” icon to view all the filter options.

 

Step-3: User can select the required options through dropdown box appearing. Upon clicking on the “search” button, the summary of Threshold limit list will be displayed for selected option.

Step-4: Click on “Reset” button to revert the changes made by selecting the options from dropdown box.

Step-5: To Print the details of the Threshold list, user need to select the “Threshold ID”, then click on “Print” button.

Step-6: User can also modify the details of the threshold by clicking on the “modify” icon in “option” field. Below page will be displayed to alert the details:

 

Step-7: A user can also delete the threshold limit record by clicking on the delete icon in “option” field. An alert message will pop-up to to warn the user about the destructive action. Upon clicking on the “OK” button, the selected record will be deleted.

 

 

Step-8: Click on “Create New” to create a new Threshold limit setting. Below page will appear where user can add the required values.

 

 

Step-9: User can also sort in ascending, descending or unsort the sorted data for the fields that select for this activity. There are sorting button available with the columns name.

 

Step-10: Click on “Triggered Alerts” within Threshold Feature. Below “Threshold Triggered Alerts” will appear.

 

Step-11: Click to “Expand” icon to view all the filter options.

 

 

Step-12: User can select the required options through dropdown box appearing. Upon clicking on the “search” button, the “Threshold Triggered Alerts” list will be displayed for selected option.

Step-13: Click on “Reset” button to revert the changes made by selecting the options from dropdown box.

Step-14: Click on the “Alert ID” and this will redirect the user to the transaction details of the transaction that caused the alert to be raised.

 

 

Step-15: Clicking on the “return to previous page” will return you to the main window page of Triggered Alerts.

Step-16: User can add the remarks/notes in the “Remarks” column by clicking on the modify icon.

 

Step-17: User can also sort in ascending, descending or unsort the sorted data for the fields that select for this activity. There are sorting button available with the columns name.

 


For Information Page Feature:

Step-1: Click on Information Page Feature, user will see the dropdown option for “Jetty Criteria”. Click on “Jetty Criteria” to load the Jetty Information page.

 

Step-2: User can select the required options through dropdown box appearing for “Terminal”. Upon clicking on the “search” button, the Jetty Information will be displayed for selected option.

Step-3: Click on “Reset” button to revert the changes made by selecting the options from dropdown box.

Step-4: Upon clicking the “Jetty No”, user will see the display of the Jetty information.

 

 

Step-5: User can also sort in ascending, descending or unsort the sorted data for the fields that select for this activity. There are sorting button available with the columns name.

 

 


For Report Feature: 

Step-1: Upon clicking on the Report Feature, user will see the Report History page.

 

Step-2: Click to “Expand” icon to view all the filter options.

 

Step-3: User can select/enter the required options through dropdown box appearing for the selection. Upon clicking on the “search” button, the Reports details will be displayed for selected option.

Step-4: Click on “Reset” button to revert the changes made by selecting the options from dropdown box.

Step-5: User can also sort in ascending, descending or unsort the sorted data for the fields that select for this activity. There are sorting button available with the columns name.

 

Step-6: User can click on the “Report Name” which is clickable link, a new page will appear with details of that report name selected. E.g., Selected Report Name is “Closed txn with Advario”

 

 

Step-7: To Print the details of the Report selected, user need to check/select the records that they want to print, then click on “Print” button.

 

Step-8: To download the details of the Report in csv, user need to select the records, then click on “CSV” button. User will be able to download the report details in csv format in their desktop.

 

Step-9: Clicking on the “return to previous page” will return you to the main window page of Report History.

Step-10: User can also see the Template Information, upon clicking on the “Template Used” record and the template information will be displayed as shown below.

 

 

Step-11: User can also delete the report record from Report History page by clicking on the delete icon on “Option” column. There will be a pop-up message to warn the user of the destructive action. Clicking on the “OK” button will delete the record.

 

Step-12: User can also create a new report by clicking on the “Create New”. Users need to select/enter the required details which is needed to create a new report.

 

 

Step-13: Clicking on “Generate Report”, will help users to generate a new report and new page will open with generated records of report.

 

Step-14: Clicking on “Save Template”, will help users to save the template.
